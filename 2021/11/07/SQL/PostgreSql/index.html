    PostgreSQL在未创建新用户时，默认登录用户是超级用户postgres。

    数据库用户操作

    1. 输入登录命令，输入正确的账号密码

      psql -U postgres -d postgres
psql -U <用户名> -d <数据库名>

    2. 创建数据库新用户

      CREATE USER Username WITH PASSWORD '*****';

    3. 创建数据库

      CREATE DATABASE <数据库名>;

    4. 授予用户数据库全部权限

      GRANT ALL PRIVILEGES ON DATABASE <数据库名> TO <用户名>;

    5. 设置年月日输入格式（在线有效，离线无效）

      SET datestyle TO 'DMY';

    6. 删除，添加，修改属性

      ALTER TABLE SC DROP COLUMN grade;
ALTER TABLE SC ADD COLUMN grade NUMERIC(3, 0);
ALTER TABLE S ALTER COLUMN sage TYPE NUMERIC(3, 0);

    7. 删除，添加约束

      ALTER TABLE S DROP CONSTRAINT "s_sage_check";
ALTER TABLE SC ADD CONSTRAINT "sc_grade_check" CHECK(grade >= 0 AND grade <= 100);
ALTER TABLE SC ADD PRIMARY KEY(sno, cno);

    数据库资源操作

    1. 显示所有表名称和结构

      命令 :

      \d
\d <TableName>

    2. 创建表

      CREATE TABLE <表名>
(<列名> <数据类型> [列级完整性约束条件]
[,<列名> <数据类型> [列级完整性约束条件]]
…
[,<表级完整性约束条件>…]);

    数据库的查询操作

    1. 查询的一般格式

      SELECT [DISTINCT] <目标列表达式列表>
FROM <基本表或视图列表>
[WHERE <条件表达式>]
[GROUP BY <列名列表> [HAVING <条件表达式>]]
[ORDER BY <列名> [ASC|DESC]];

    数据库的增删改操作

    1. 插入元组：

      • 不能违背完整性约束！

      • 如果没有指定元组某个元素的值，系统默认将其赋值为空。

      • 如果表明后添加(属性)，那么属性列的顺序可与表中不一样，只要和常量相对应即可

      语法和示例

      INSERT INTO <表名>[(<属性列1> [,<属性列2>]…)]<子查询>;
INSERT INTO S(Sno, Sname, Ssex, Sage) VALUES ('S14', 'Tom', 'M');
INSERT INTO S(Sno, Sname, Ssex) VALUES ('S14', 'Tom', 'M');

    2. 插入子查询结果：

      例：计算每门课的平均分，存入如下数据库表中

      CREATE TABLE AvgGrade(
   Cno CHAR(9) PRIMARY KEY,
   grade INT;
);

INSERT INTO AvgGrade
   SELECT Cno, AVG(grade)
   FROM SC
   GROUP BY Cno;

    3. 修改内容

      语法和示例

      UPDATE <表名>
SET <列名1>=<表达式1> [, <列名2>=<表达式2>]…
[WHERE <条件表达式>];

UPDATE SC
SET grade = grade * 1.3
WHERE Cno IN
   (SELECT Cno FROM C WHERE Cname = 'DB')

    4. 删除内容

      语法和示例

      DELETE FROM <表名> [WHERE <条件表达式>];

DELETE FROM SC
WHERE Cno IN (SELECT Cno FROM C
WHERE Cname='DB');

    数据库理论知识

    一， 数据库的空值

    1. 一般需特殊处理，常用的空值处理方法：NULLIF(p, v) : 若p非空值，返回p，否则返回v
    2. 空值与其它值的算术运算结果为空值，如：NULL+3=NULL
    3. 空值进行比较的结果为UNKNOWN。
    4. UNKNOWN可进行逻辑运算
      • 如：3>NULL的结果不为真也不为假，结果为UNKNOWN
      • 如：UNKNOWN OR TRUE=TRUE、UNKNOWN AND TRUE=UNKNOWN

    二， 数据库视图

    1. 视图

      可以理解成数据库的“窗口”，是一个“虚表”，只能看到视图定义的信息，数据实际存放在基本表中。

      在视图中的增删改相当于对数据的增删改。

    2. 创建视图：

      语法和示例

      CREATE [OR REPLACE] VIEW
<视图名> [(<列名1>[, <列名2>]…)]
AS <子查询>
[WITH CHECK OPTION]; /* 防止插入非法数据 */

    3. 删除视图

      DROP VIEW <视图名> [CASCADE];

DROP VIEW M_C1 CASCADE CONSTRAINT;
      • 如选择了CASCADE，则级联删除由该视图导出的其它视图。
      • 若未选择，则保留由其导出的视图，但这些导出的视图将无法使用，除非重新定义该视图。
      • SQL没有提供修改视图的方法，修改需要先删除后创建。

    4. 在视图中查询和更新：与基本表的查询和更新相同

      • 目前多数DBMS对行列子集视图的查询能够进行正确转换，但对非行列子集视图的查询不一定能够转换

      • 一些的视图是不可更新的，因为对这些视图的更新不能唯一地、有意义地转换成对相应基本表的更新

    5. 视图种类：

      1. 行列子集视图
      CREATE OR REPLACE VIEW
M_S(Sno, Sname, Ssex, Sage)
AS SELECT Sno, Sname, Ssex, Sage
FROM S
WHERE Ssex='M'
WITH CHECK OPTION;
      1. 基于多表的视图
      CREATE OR REPLACE VIEW
M_C1
AS SELECT S.Sno, Sname, grade
FROM S, SC
WHERE S.Sno=SC.Sno AND
S.Ssex='M' AND
SC.Cno='C1';
      1. 基于视图的视图
      CREATE OR REPLACE VIEW M_C1_A
AS SELECT Sno, Sname, grade
FROM M_C1
WHERE grade>=90;
      1. 带表达式的视图
      CREATE OR REPLACE VIEW
Birth_S
AS SELECT Sno, Sname, Ssex, 2021-Sage Sbirth
FROM S;
      1. 分组视图
      CREATE OR REPLACE VIEW
S_G(Sno, Gavg)
AS SELECT Sno, AVG(grade)
FROM SC
GROUP BY Sno
ORDER BY Sno ASC;

    6. 为什么需要视图？

      1. 简化查询操作，一次定义，多次使用。

      2. 分别满足不同用户的需求，使不同用户从不同角度看待同一数据

      3. 提供逻辑独立性

      数据库模式对应关系：模式 \(\stackrel{对应}{\rightarrow}\) 表，外模式 \(\stackrel{对应}{\rightarrow}\) 视图

      应用程序使用外模式访问数据库。如果改变了表，那么只需改变模式与外模式的映射，这样应用程序就不用发生改变。

      1. 对机密数据提供安全保护

      2. 适当利用视图可以更清晰地表达查询

    三，触发器

    1. 触发器：一种完整性控制机制

    2. 定义触发器：不同数据库略有不同

    3. 应用：对外码的修改

      举例：在表T上定义触发器，如修改教师工号，则级联修改C表中的教师工号。

      CREATE OR REPLACE TRIGGER t1
AFTER -- 触发时机
UPDATE OF Tno ON T -- 触发事件
FOR EACH ROW -- 触发器类型：行级触发器
BEGIN -- 触发器执行的动作
   UPDATE C SET Tno=:NEW.Tno
   WHERE Tno=:OLD.Tno
END;

      注释：

      • Before：触发事件执行之前调用触发器。

      • After：触发事件执行之后调用触发器。

      • NEW：数据类型是RECORD。对于行级触发器，它存有INSERT或UPDATE操作产生的新的数据行。对于语句级触发器，它的值是NULL。

      • OLD：数据类型是RECORD。对于行级触发器，它存有被UPDATE或DELETE操作修改或删除的旧的数据行。对于语句级触发器，它的值是NULL。
