    并行式集成学习

    特点：仍然使用审核文章的例子。并行式集成学习相当于把一篇文章同时分给所有人审核，所有人审核完后，将审核结果结合到一起，众人一起商议，通过投票等方式，可以尽量避免审核的错误，从而正确选出应该改正的错误。

    代表性思路：Bagging

    代表性方法：随机森林

    相对于串行的好处

    1. 求解个体学习器时是并行的，加快了学习效率。
    2. 每个学习器都在完成自己的工作，不考虑其它学习器的影响，与整体目标的偏差大，方差小。

    Bagging算法过程

    输入：

    • 训练集\(D = \{(\pmb{x}_{1},y_{1}),(\pmb{x}_{2},y_{2}),\dots,(\pmb{x}_{m},y_{m})\}\)

    • 基学习算法\(\mathfrak{L}\)

    • 训练轮数\(T\)

    过程：

    1. \(\textbf{for}\ \ t = 1 \dots T \ \ \textbf{do}\)
    2. \(\ \ h_{t}=\mathfrak{L}(D,\mathcal{D}_{t});\)
    3. \(\textbf{end}\ \ \textbf{for}\)

    输出：\(H(\pmb{x})=\underset{y\in \mathcal{Y}}{\arg\max}\stackrel{T}{\underset{t=1}{\sum}}\mathbb{I}(h_{t}(\pmb{x})=y)\)

    Bagging的样本子集如何求得

    为了保证差异性，可以使用自主采样法(bootstrap sampling)，即有放回的随机抽取。如果抽的次数足够多，没有抽到的样本占比大致为\(\underset{n\rightarrow\infty}{\lim}=(1-\frac{1}{n})^n=\frac{1}{e}\approx 36.6\%\).正好可以使用这些没有抽到的样本作为验证集进行包外估计。

    随机森林

    在Bagging的基础上，进一步在决策树的训练过程中引入了随机选择。假设样本包含\(d\)个属性，对基决策树的每个节点，先从该节点的属性集合中随机选择一个包含\(k(0<k\leq d)\)个属性的子集，然后再从这个子集中选择一个最优属性用于划分。一般情况下推荐\(k=\log_2d\)。

    随机森林训练效率通常优于Bagging，因为每个节点的划分只需要部分属性参与，而随机森林的泛化误差通常低于Bagging，因为属性的扰动为每个基决策树提供了更好的泛化性能。

    聚合意见的方法——结合策略

    学习器结合的好处

    1. 从统计的方面来看，由于学习任务的假设空间往往很大，可能有多个假设在训练集上达到相同性能，此时使用单个学习器可能因误选导致泛化性能不佳，结合多个学习器会减少这一风险。
    2. 从计算的方面来看，学习算法往往会陷入局部极小，有时局部极小点对应的泛化性能很糟，而通过多次运行之后进行结合，可降低进入局部极小点的风险。
    3. 从表示的方面来看，某些学习任务的真实假设可能不在当前学习算法所考虑的假设空间中，此时使用单学习器一定无效，而通过结合多学习器，由于相应假设空间有所扩大，有可能学得更好的近似。

    一，平均法

    • 简单平均法，是加权平均法\(w_i=1/T\)时的特例。 \[ H(\pmb{x})=\stackrel{T}{\underset{i=1}\sum}h_i(\pmb{x}) \]

    • 加权平均法 \[ H(\pmb{x})=\stackrel{T}{\underset{i=1}\sum}w_i h_i(\pmb{x}) \]

    一般而言，在个体学习器性能相差较大时使用加权平均法。

    二，投票法

    为方便讨论，将\(h_i\)在样本\(\pmb{x}\)上的预测输出表示为一个\(N\)维向量\((h_i^1(\pmb{x});h_i^2(\pmb{x});\dots;h_i^N(\pmb{x}))\)，其中\(h_i^j(\pmb{x})\)是\(h_i\)在类别标记\(c_j\)上的输出。

    • 绝对多数投票法，即若某标记得票超过半数，即预测为该标记，否则拒绝预测。 \[ H(\pmb{x})=\begin{cases}c_j,\textbf{if}\ \stackrel{T}{\underset{i=1}\sum}h_i^j(\pmb{x})>0.5\stackrel{N}{\underset{k=1}\sum}\stackrel{T}{\underset{i=1}\sum}h_i^k(\pmb{x}) \\ reject, \textbf{otherwise}\end{cases} \]

    • 相对多数投票法，即预测为得票最多的标记，若同时有多个标记获得最高票，则随机选择一个。 \[ H(\pmb{x})=c_{\underset{j}{\arg\max}\stackrel{T}{\underset{i=1}\sum}h_i^j(\pmb{x})} \]

    • 加权投票法，与加权平均法类似，通常\(w_i\geq 0\)，\(\stackrel{T}{\underset{i=1}\sum}w_i=1\) \[ H(\pmb{x})=c_{\underset{j}{\arg\max}\stackrel{T}{\underset{i=1}\sum}w_ih_i^j(\pmb{x})} \]

    三，学习法

    当训练数据很多时，一种更为强大的结合策略是使用“学习法”，即通过另一个学习器来结合。

    学习法的代表方法是Stacking。Stacking先从初始数据集训练出初级学习器，然后“生成”一个新数据集用于训练次级学习器，在这个新数据集中，初级学习器的输出被当作样例输入特征，而初始样本的标记仍被当作样例标记。下面是Stacking算法的伪代码描述：

    输入：

    • 训练集\(D = \{(\pmb{x}_{1},y_{1}),(\pmb{x}_{2},y_{2}),\dots,(\pmb{x}_{m},y_{m})\}\)

    • 初级学习算法\(\mathfrak{L}_1,\mathfrak{L}_2,\dots,\mathfrak{L}_T\)

    • 次级学习算法\(\mathfrak{L}\)

    过程：

    1. \(\textbf{for}\ \ t = 1 \dots T \ \ \textbf{do}\)
    2. \(\ \ h_{t}=\mathfrak{L}_t(D);\)
    3. \(\textbf{end}\ \ \textbf{for}\)
    4. \(D' = \varnothing\)
    5. \(\textbf{for}\ \ i = 1 \dots m \ \ \textbf{do}\)
    6. \(\ \ \textbf{for}\ \ t = 1 \dots T \ \ \textbf{do}\)
    7. \(\ \ \ \ z_{it}=h_t(\pmb{x})\)
    8. \(\ \ \textbf{end for}\)
    9. \(\ \ D'=D'\cup((z_{i1},z_{i2},\dots,z_{iT}),y_i)\)
    10. \(\textbf{end for}\)
    11. \(h'=\mathfrak{L}(D')\)

    输出：\(H(\pmb{x})=h'(h_1(\pmb{x}),h_2(\pmb{x}),\dots,h_T(\pmb{x}))\)

    多样性增强的方法

    1. 数据样本扰动

      • 利用不同的数据子集训练出不同的个体学习器，数据样本扰动通常是基于采样法，例如在Bagging的自助采样，AdaBoosting中的序列采样。

      • 对输入扰动敏感的基学习器效果较好：决策树，神经网络等

      • 对输入扰动不敏感的基学习器效果不好：线性学习器，SVM，朴素贝叶斯，k近邻等

    2. 输入属性扰动

      • 从初始属性集中抽取出若干个属性子集，再基于每个属性子集训练一个基学习器，算法描述如下

      输入：

      • 训练集\(D = \{(\pmb{x}_{1},y_{1}),(\pmb{x}_{2},y_{2}),\dots,(\pmb{x}_{m},y_{m})\}\)

      • 基学习算法\(\mathfrak{L}\)

      • 基学习器数\(T\)

      • 子空间属性数\(d'(d'<d)\)

      过程：

      1. \(\textbf{for}\ \ t = 1 \dots T \ \ \textbf{do}\)

      2. \(\ \ \mathcal{F}_{t}=\text{RS}(D',d');\)1

      3. \(\ \ D_{t}=\text{Map}_{\mathcal{F}_t}(D);\)

      4. \(\ \ h_t=\mathfrak{L}(D_t)\)

      5. \(\textbf{end}\ \ \textbf{for}\)

      输出：\(H(\pmb{x})=\underset{y\in \mathcal{Y}}{\arg\max}\stackrel{T}{\underset{t=1}\sum}\mathbb{I}(h_t(\text{Map}_{\mathcal{F}_t}(\pmb{x})=y))\)

      • 对包含属性较少的数据效果不好。

    3. 输出属性扰动

      • 随机改变一些训练样本的类标记
      • 对输出表示进行转化

    4. 算法参数扰动

      • 设置算法的参数，例如神经网络的隐层神经元数，初始连接权值等，通过随机设置不同的参数，往往可以产生差别较大的个体学习器。
